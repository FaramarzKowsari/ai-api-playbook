# GitHub-to-Zenodo release checklist

1. Create the public GitHub repository and push the verified source tree.
2. Confirm `CITATION.cff`, `LICENSE`, author name, ORCID, version, and release date.
3. Connect the repository in Zenodo's GitHub integration.
4. Reserve or mint a DOI for the software release.
5. Replace any DOI placeholder in the README and citation metadata.
6. Create a signed or annotated tag: `v1.0.0`.
7. Publish a GitHub Release from that tag with a concise changelog.
8. Confirm Zenodo archived the correct tag and files.
9. Add the DOI badge and canonical DOI link to the README.
10. Create a separate Zenodo publication record for the book PDF and link both records with Related Identifiers.
